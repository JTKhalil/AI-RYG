# Cursor AI 信号灯 — 使用说明

本文档介绍如何通过 ESP32-C3 与四线红绿灯模块，实时显示 Cursor Agent 的工作状态。

---

## 一、项目简介

当你在 Cursor 中使用 Agent（AI 助手）时，信号灯会自动切换颜色：

| 灯光 | 含义 |
|------|------|
| **黄灯** | 思考中（提交问题、推理、调用工具） |
| **绿灯** | 思考结束（本轮 Agent 回复完成） |
| **红灯** | 报错（工具调用失败，红灯慢闪） |
| **熄灭** | 会话结束 |

### 工作原理

```
你在 Cursor 发送消息
        │
        ▼
Cursor Hooks 监听 Agent 事件
        │
        ▼
Python 脚本 (hook_bridge.py)
        │
        ▼
USB 串口 (COM14 等)
        │
        ▼
ESP32-C3 固件解析命令
        │
        ▼
红绿灯模块亮对应颜色
```

Cursor 本身不提供「信号灯 API」，因此通过 **Cursor Hooks** 在 Agent 各阶段自动调用 Python 脚本，经串口控制 ESP32。

## 二、所需硬件

| 物品 | 说明 |
|------|------|
| ESP32-C3 开发板 | USB 直连电脑，用于串口通信 |
| 四线红绿灯模块 | 引脚：`PIN1`、`PIN2`、`VCC`、`GND` |
| USB 数据线 | 连接 ESP32 与电脑 |
| 杜邦线（可选） | 若模块未焊接排针 |

### ESP32-C3 可用引脚

板子通常引出：**GPIO 0–10、20–21**，以及 **3V / 5V / GND**。

以下引脚建议**不要**接 LED 或外设：

| 引脚 | 原因 |
|------|------|
| GPIO 0 | BOOT / 下载按钮 |
| GPIO 2、8、9 | 启动配置脚 |
| GPIO 20、21 | USB 串口通信（与电脑通信需保留） |

## 三、硬件接线

### 四线模块接线表

| 模块上的线 | 接到 ESP32-C3 |
|-----------|---------------|
| **VCC** | **5V**（模块标注 3.3V 时改接 3V） |
| **GND** | **GND** |
| **PIN1** | **GPIO 4** |
| **PIN2** | **GPIO 5** |

### 颜色编码（本模块实测）

两根信号线组合控制三种颜色：

| PIN1 | PIN2 | 亮灯 |
|------|------|------|
| 0 | 0 | 全灭 |
| 0 | 1 | **红灯** |
| 1 | 0 | **黄灯** |
| 1 | 1 | **绿灯** |

> **注意**：PIN1 必须接 GPIO 4，PIN2 必须接 GPIO 5，不要对调模块上的两根信号线。

### 接线示意

```
  红绿灯模块                    ESP32-C3
  ┌──────────┐                ┌──────────┐
  │   VCC    │───────────────►│   5V     │
  │   GND    │───────────────►│   GND    │
  │   PIN1   │───────────────►│  GPIO 4  │
  │   PIN2   │───────────────►│  GPIO 5  │
  └──────────┘                └──────────┘
                                      │
                                      │ USB
                                      ▼
                                    电脑
```

## 五、安装步骤

### 步骤 1：烧录 ESP32 固件

1. 打开 Arduino IDE
2. **文件 → 打开** → 选择项目中的  
   `esp32/ai_traffic_light/ai_traffic_light.ino`
3. **工具** 菜单设置：
   - 开发板：**ESP32C3 Dev Module**
   - USB CDC On Boot：**Enabled**（USB 直连时必选）
   - 端口：选择 ESP32 对应的 COM 口
4. 点击 **上传**
5. 打开 **串口监视器**（波特率 115200），应看到：

   ```
   Cursor AI Traffic Light Ready (ESP32-C3)
   Commands: Y=thinking G=done R=error O=off
   ```

> 烧录完成后请**关闭串口监视器**，否则 PC 端脚本无法占用同一 COM 口。

---

### 步骤 2：安装 Python 依赖

在项目目录打开 PowerShell：

```powershell
cd D:\ESP32\ai信号灯
```

安装依赖：

```powershell
& "$env:LOCALAPPDATA\Programs\Python\Python312\python.exe" -m pip install -r pc\requirements.txt
```

若 Python 已加入系统 PATH，也可使用：

```powershell
pip install -r pc\requirements.txt
```

---

### 步骤 3：配置串口

1. 打开 **设备管理器 → 端口 (COM 和 LPT)**，确认 ESP32 的 COM 号（例如 `COM14`）
2. 编辑 `pc/config.json`：

```json
{
  "port": "COM14",
  "baud": 115200,
  "log_file": ""
}
```

将 `"port"` 改为你的实际 COM 口。

---

### 步骤 4：测试硬件与串口

运行自动测试脚本（依次：熄灭 → 黄 → 绿 → 红 → 熄灭，每步约 2 秒）：

```powershell
& "$env:LOCALAPPDATA\Programs\Python\Python312\python.exe" "D:\ESP32\ai信号灯\pc\test_serial.py"
```

也可手动发送单个状态：

```powershell
# 黄灯（思考中）
& "$env:LOCALAPPDATA\Programs\Python\Python312\python.exe" "D:\ESP32\ai信号灯\pc\serial_sender.py" thinking

# 绿灯（完成）
& "$env:LOCALAPPDATA\Programs\Python\Python312\python.exe" "D:\ESP32\ai信号灯\pc\serial_sender.py" done

# 红灯（报错）
& "$env:LOCALAPPDATA\Programs\Python\Python312\python.exe" "D:\ESP32\ai信号灯\pc\serial_sender.py" error

# 熄灭
& "$env:LOCALAPPDATA\Programs\Python\Python312\python.exe" "D:\ESP32\ai信号灯\pc\serial_sender.py" off
```

**测试通过标准**：灯光顺序为 **熄灭 → 黄 → 绿 → 红 → 熄灭**。

---

### 步骤 5：安装 Cursor Hooks

一键安装（将 `-Port` 改为你的 COM 口）：

```powershell
powershell -ExecutionPolicy Bypass -File "D:\ESP32\ai信号灯\scripts\setup_hooks.ps1" -Port COM14
```

脚本会：

1. 检查 Python 与 pyserial
2. 写入 / 合并 `%USERPROFILE%\.cursor\hooks.json`
3. 配置 Hook 在 Agent 各阶段调用 `hook_bridge.py`

**安装完成后必须重启 Cursor**，Hook 才会生效。

## 六、日常使用

1. ESP32 通过 USB 连接电脑并保持上电
2. 打开 Cursor，进入 Agent 对话
3. 发送消息后 → **黄灯**亮起
4. Agent 回复完成 → **绿灯**
5. 若工具执行失败 → **红灯**慢闪
6. 关闭 Agent 会话 → 灯**熄灭**

### 在 Cursor 中调试 Hook

- 打开 **Cursor 设置 → Hooks**，查看 Hook 是否已加载
- 打开 **输出面板 → Hooks** 通道，查看 Hook 执行日志
- 若 Hook 报错，通常是 Python 路径或 COM 口配置问题

---

## 七、Hook 事件说明

| Cursor 事件 | 触发时机 | 灯光 |
|-------------|----------|------|
| `beforeSubmitPrompt` | 你提交问题 | 黄 |
| `afterAgentThought` | Agent 推理中 | 黄 |
| `preToolUse` | 即将调用工具 | 黄 |
| `postToolUse` | 工具调用完成（仍在工作中） | 黄 |
| `subagentStart` | 子 Agent 启动 | 黄 |
| `postToolUseFailure` | 工具调用失败 | 红 |
| `stop` | 本轮 Agent 结束 | 绿 |
| `sessionEnd` | 会话结束 | 熄灭 |

> Agent 多步调用工具时，黄灯会持续亮较长时间，直到 `stop` 事件触发后才变绿，这是正常现象。

---

## 八、串口通信协议

- **波特率**：115200
- **格式**：单字符 + 换行（`\n`）

| 命令 | 含义 | 模块 PIN1/PIN2 |
|------|------|----------------|
| `Y` | 黄灯 | 1 / 0 |
| `G` | 绿灯 | 1 / 1 |
| `R` | 红灯（慢闪） | 0 / 1 |
| `O` | 熄灭 | 0 / 0 |

ESP32 收到命令后会回复 `OK:YELLOW`、`OK:GREEN` 等（调试用，Hook 不依赖此回复）。

## 十、常见问题

### 1. 测试时颜色顺序不对

- 确认已烧录**最新固件**
- 确认 PIN1 → GPIO 4，PIN2 → GPIO 5，未接反
- 若仍不对，对照本文档第三节编码表检查

### 2. 串口打不开 / Permission denied

- 关闭 Arduino **串口监视器**
- 确认没有其他程序占用 COM 口
- 重新插拔 USB，在设备管理器中确认 COM 号是否变化

### 3. Hook 不触发、灯无反应

- **重启 Cursor**
- 检查 `%USERPROFILE%\.cursor\hooks.json` 是否存在
- 确认 Hook 命令中的 Python 路径正确，例如：  
  `C:\Users\clear\AppData\Local\Programs\Python\Python312\python.exe`
- 在 Hooks 输出通道查看错误信息

### 4. 一直是黄灯不变绿

- 多步 Agent 任务期间保持黄灯是正常的
- 只有本轮完全结束（`stop` 事件）才会变绿
- 若长时间不变，可能是 Agent 仍在运行或 Hook 未触发 `stop`

### 5. Python 找不到

Windows 可能只有 Microsoft Store 占位符。请使用完整路径：

```powershell
& "$env:LOCALAPPDATA\Programs\Python\Python312\python.exe" --version
```

或从 [python.org](https://www.python.org/downloads/) 安装，并勾选 **Add Python to PATH**。

### 6. 更换 COM 口

修改 `pc/config.json` 中的 `port`，无需重新烧录固件。若已安装 Hook，可重新运行：

```powershell
powershell -ExecutionPolicy Bypass -File "D:\ESP32\ai信号灯\scripts\setup_hooks.ps1" -Port COMx
```

## 十一、自定义与扩展

### 更换 GPIO 引脚

编辑 `esp32/ai_traffic_light/ai_traffic_light.ino`：

```cpp
#define PIN1  4   // 模块 PIN1 所接 GPIO
#define PIN2  5   // 模块 PIN2 所接 GPIO
```

修改后需重新烧录。

### 修改颜色编码

在固件 `applyState()` 函数中调整 `applyTwoPin(高/低, 高/低)` 的组合，与模块实际行为对应。

### 卸载 Hook

编辑 `%USERPROFILE%\.cursor\hooks.json`，删除与 `hook_bridge.py` 相关的条目，保存后重启 Cursor。

## 十二、Windows EXE 桌面程序（推荐）

可将 Python 程序打包为独立 EXE，双击运行后：

- 自动启动串口守护（监听 Cursor 状态）
- 自动安装 Cursor Hooks
- 在系统托盘显示图标（右键可管理）

### 打包

```powershell
powershell -ExecutionPolicy Bypass -File "D:\ESP32\ai信号灯\scripts\build_exe.ps1"
```

生成：`pc\dist\CursorTrafficLight.exe`

### 安装到本机

```powershell
powershell -ExecutionPolicy Bypass -File "D:\ESP32\ai信号灯\scripts\install_app.ps1"
```

会安装到：

| 项目 | 路径 |
|------|------|
| 程序 | `%LOCALAPPDATA%\CursorTrafficLight\CursorTrafficLight.exe` |
| 配置 | `%APPDATA%\CursorTrafficLight\config.json` |
| 日志 | `%APPDATA%\CursorTrafficLight\app.log` |

桌面会创建快捷方式 **Cursor AI 信号灯**。

### 使用

1. 双击运行 EXE（托盘出现黄色图标）
2. **重启 Cursor**
3. ESP32 保持 USB 连接

托盘菜单：

- **安装 Cursor Hooks** — 重新写入 Hook 配置
- **打开配置目录** — 修改 COM 口等
- **开启开机自启** — 登录 Windows 后自动运行
- **退出** — 停止监听

### 命令行模式（供 Hook 调用，无需手动运行）

```text
CursorTrafficLight.exe --hook thinking
CursorTrafficLight.exe --hook done
CursorTrafficLight.exe --install-hooks
```

## 附录：三线独立 LED 模块（非四线模块）

若你的模块是红 / 黄 / 绿各一根信号线 + GND（共 4 线但 3 路独立控制），需修改固件：

1. 将 `TWO_PIN_MODULE` 改为 `false`
2. 接线：红→GPIO4，黄→GPIO5，绿→GPIO6，GND→GND
3. 重新烧录
